# FusionTwin Principles

> Build the foundation, not the skyscraper.

## Our North Star

FusionTwin is not just a fusion simulator—it is a platform for building, exploring, understanding, and eventually optimizing fusion reactors.

## Core Principles

### Data First

Everything possible should be defined as data rather than hard-coded.

- Reactors
- Materials
- Diagnostics
- Experiments
- Scenarios

If it can be expressed as data, it should be.

---

### Modular Everything

Every major subsystem should be independently replaceable.

- Geometry
- Rendering
- Physics
- Diagnostics
- AI
- Education

Small modules. Clean interfaces. Minimal coupling.

---

### Beautiful by Default

Scientists stare at these models for hours.

Students need to be inspired.

FusionTwin should be visually compelling from day one.

---

### Educational Without Being Simplified

Never dumb the science down.

Instead, reveal complexity progressively.

Beginners should feel welcome.
Experts should never feel constrained.

---

### Open Core

FusionTwin should be extensible.

Community contributions should include:

- Reactor definitions
- Diagnostics
- Experiments
- Plugins
- Visualization modules

The platform should grow beyond its original creators.

---

## Design Philosophy

1. Build the foundation before the skyscraper.
2. Architecture before optimization.
3. Visualization before high-fidelity physics.
4. Data before code.
5. Incremental progress every sprint.

---

## Long-Term Vision

FusionTwin aims to become the **Unreal Engine for fusion reactors**—a platform where students, researchers, and engineers can design, compare, simulate, and understand fusion systems through a shared, extensible ecosystem.



## The Sprint Rule

Every sprint should leave FusionTwin in a demonstrably better state.

A sprint is successful if it delivers at least one of the following:

- A new capability
- A cleaner architecture
- Better documentation
- Improved visualization
- A deeper educational experience

Small, consistent progress compounds into extraordinary software.
